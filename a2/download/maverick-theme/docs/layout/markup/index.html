Grids

The theme is fluid layout with 12 columns. The sidebar (on the left) and the infobar (right) are removed from the grid.

General Markup

The general template structure is the same throughout the template. Here is the general structure.

<!DOCTYPE html>
<html lang="en">
<head><!--insert all stylesheet links --></head>
<body>
    <header id="topnav" class="navbar">
        <!-- All header links, including sidebar triggers and drop downs-->
    </header>
    <nav id="headernav" class="navbar">
        <!-- The horizontal nav links are generated here -->
    </nav>

    <div id="wrapper">
        <div id="layout-static">
            <div class="static-sidebar-wrapper">
                <nav class="static-sidebar" role="navigation">
                    <!-- All sidebar links are generated here -->
                </nav>
            </div>
            <div class="static-content-wrapper">
                <div class="static-content">
                    <div id="wrap" class="mainview-animation animated">
                        <!-- The actual contents of the page -->
                    </div> <!--wrap -->
                </div>
                <footer role="contentinfo">
                    <!-- Footer content -->
                </footer>
            </div>
        </div>
    </div>
</body>
</html>

The contents of the infobar and the header has been separated from the app/index.html page to reduce clutter. They can be found at the app\views\layout folder.

Partial Views

Wildcard routes look for a .html file with the name matching the URL segment after the #/.

So, when you hit the URL #/layout-fixed, Maverick will make a request to the server for layout-fixed.html partial view file located in app/views/.

body classes

Several important classes are added to the body element to control overall page appearance. They are added conditionally by the controller (MainController) using the $theme service (more on this here).

<body
  ng-app="themesApp"
  ng-controller="MainController"
  class="{{getLayoutOption('sidebarThemeClass')}} {{getLayoutOption('topNavThemeClass')}}"
  ng-class="{
              'static-header': !getLayoutOption('fixedHeader'),
              'focusedform': getLayoutOption('fullscreen'),
              'layout-horizontal': getLayoutOption('layoutHorizontal'),
              'fixed-layout': getLayoutOption('layoutBoxed'),
              'sidebar-collapsed': getLayoutOption('leftbarCollapsed') && !getLayoutOption('leftbarShown'),
              'show-infobar': getLayoutOption('rightbarCollapsed'),
              'show-sidebar': getLayoutOption('leftbarShown')
            }"
  ng-click="hideHeaderBar();hideChatBox()"
  to-top-on-load
>
<!-- other stuff -->
</body>

As theme options change, so do these classes. Check out the $theme service documentation for details.

Boxed Layouts

In JavaScript, you can use the $theme service to turn on boxed layout with $theme.set('layoutBoxed', true).

You can change the width of the boxed layout and the background from the variables.less file, @boxed-layout.

Fullscreen Mode

You can turn on full screen mode with $theme.set('fullscreen', true).

The body needs a focusedform class if using the site for pages such as log in or sign up which focuses on a single form in the page.

Horizontal Layout

To turn on horizontal layout, use $theme.set('layoutHorizontal', true).

The body needs a layout-horizontal class to use the horizontal layout.

Navigation

Maverick has a data-driven navigation system in keeping with the Angular methodology. Therefore, it's split between view and model parts.

First, the view is set up to recursively render the menu items using the templates/nav_renderer.html template snippet.

<ul ng-controller="NavigationController" id="sidebar" sticky-scroll="40">
  <li ng-repeat="item in menu"
      ng-class="{ hasChild: (item.children!==undefined),
                    active: item.selected,
                      open: (item.children!==undefined) && item.open,
           'nav-separator': item.separator==true,
            'search-focus': (searchQuery.length>0 && item.selected) }" 
        ng-show="!(searchQuery.length && !item.selected)"
      ng-include="'templates/nav_renderer.html'"
    ></li>
</ul>

Note that the behavior is driven by the NavigationController controller. This is defined in the app/scripts/controllers/Navigation.js file (more on this here.
