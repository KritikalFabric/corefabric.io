Installation

Requirements

Minimum:

  • HTTP Server

Other (required for production):

  • Node/NPM
  • Bower
  • Grunt

If you don't have Node and/or NPM installed, please visit this page for guidance.

Note On *nix systems, you may need to prefix the commands given on this page with sudo.

Quick installation

The easiest way to get Maverick up and running is with the prebuilt folder we've provided.

First you'll need an HTTP server. If you're on Windows, check out XAMPP or MAMP for Mac.

Once you've installed an Apache server, just paste the prebuilt in the htdocs folder. Now visit http://localhost/prebuilt/ and Maverick should be up and running!

Bower

Maverick uses Bower for front-end package management. (If you wish not to use Bower, use the prebuilt folder.)

First install Bower from the command line.

npm install -g bower

Now install Bower dependencies.

bower install

Your Bower packages should be installed in the app/bower_components folder.

Note The current directory should be the root folder while running these commands.

Grunt

Install grunt with the following command:

npm install -g grunt-cli

Now run the following command to install necessary node packages.

npm install

Note The current directory should be the root folder while running these commands.

Build for production

When you have done customizing the theme and want to put it in production, you should "build" the theme to optimize it for performance.

To do this, all you have to run is the following command:

grunt build

Note The current directory should be the root folder while running these commands.

Individual Grunt tasks

Sometimes during development you might need to run certain grunt tasks individually. These are the 2 common uses:

bowerInstall

After you have installed a new Bower package or uninstalled one, please run the following command to inject the file links into the index.html file:

grunt bowerInstall

ngtemplates

After you have updated a template file within views/templates folder, run the following command to cache the templates:

grunt ngtemplates

More on this here.

Note The current directory should be the root folder while running these commands.
